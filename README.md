## Retail Store Analysis using SQL
## Project Overview

This project demonstrates the design and analysis of a Retail Store Management Database using MySQL. The database was created to simulate a real-world retail business, managing customers, products, orders, payments, and product reviews.

The project focuses on writing business-oriented SQL queries to extract actionable insights from transactional data while demonstrating core SQL concepts such as joins, aggregations, filtering, subqueries, and set operations.

## Objectives

Design a relational database for a retail business.

Build relationships using primary and foreign keys.

Perform SQL-based business analysis.

Answer real-world business questions using SQL.

Strengthen practical database querying skills.

## Database Schema

The database consists of six related tables:

Customers – Customer information and contact details

Products – Product catalog, category, pricing, and stock

Orders – Customer order details

Order Items – Products purchased within each order

Payments – Payment records and payment methods

Product Reviews – Customer ratings and product feedback

The tables are connected through Primary Key and Foreign Key relationships to maintain referential integrity.
